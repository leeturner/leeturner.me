Overriding a Parent Dependency in Maven

PUBLISHED ON JAN 8, 2022

This weekend I am spending some time taking a look at my Snyk Dashboard and upgrading a few dependencies in old projects

The main thing I wanted to take a look at was the recent RCE (Remote Code Execution) vulnerability that was disclosed in the h2 database. This is a similar JNDI exploit to the one that impacted Log4j in December last year (called Log4Shell). You can read more about Log4Shell on the Snyk resources page and find details of the recent h2 RCE on the Snyk Vulnerability Database

As you can see from the dependency scan, I am using version 1.4.200 and this dependency is introduced via the spring-boot-starter-parent version 2.5.4

1. Update the Parent Dependencies

The first thing to do is to make sure we are using the latest version of the parent. This won’t actually fix the h2 issue given Spring Boot 2.6.2 was released before the h2 vulnerability was disclosed but using the latest parent will update a load of other dependencies as well.

  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.6.2</version>
    <relativePath/>
  </parent>

2. Override the H2 dependency

The next step is to override the h2 dependency provided by Sprint Boot and upgrade it to the version that fixes RCE vulnerability - 2.0.206.

To do this we simply add the version of h2 we want to the <properties> element in the pom.xml:

<properties>
    <h2.version>2.0.206</h2.version>
</properties>

If you use an IDE like IntelliJ you should now be able to see the updated dependency in your dependencies’ manager:

Or, if you want to do it on the command line mvn dependency:tree is your friend:
